Graph Animations

Since our focus is animation, i will only focus on Dr. B's transitions. Note that the code being referred to in the paragraph is green in color.
In the reading, we learn that adding transitions to changing visualizations is extremely simple using D3. With just a few lines of simple code we can make it possible. For example, simply chaining ".transition()" to our D3 algorithm can make the most basic transition happen for a changing visualization. 

 svg.selectAll("rect")
.data(dataset)
.transition()
.attr("y", function(d) {
return h - yScale(d);
})
.attr("height", function(d) {
return yScale(d);
})
.attr("fill", "red");

If we wanted to specify how long the transition period takes, we could simply add a ".duration()" line as shown below: 

    svg.selectAll("rect")
.data(dataset)
.transition().duration(1000)
.attr("y", function(d) {
return h - yScale(d);
})
.attr("height", function(d) {
return yScale(d);
})
.attr("fill", "red");

Note that the duration is in miniseconds (ms), so the value 1000 represents 1 second.
To determine "how" the transition happens(how will the animation behave in terms of it's speed of changing as opposed to duration), we can use an ease() line.

    .transition()
.duration(2000)
.ease(d3.easeLinear)

To make something happen at the start or end of a transition we can simply use a .on() function as shown: 

    .on("start", function() {
d3.select(this)
.transition()
.duration(250)
.attr("fill", "brown")
.attr("r", 3);
})

"start" would execute the function that comes after, at the beginning of the transition. "end" would execute it at the end.
